# AWS Identity and Access Management

AWS IAM is AWS's foundational identity and access control service. It defines who or what can access AWS resources and under what conditions.

## Core Responsibilities

- Identity and role management
- Policy-based authorization
- Temporary credential workflows
- Least-privilege enforcement and access analysis

## Why It Matters

- Every other AWS service depends on IAM for secure access control
- It is central to multi-account governance and workload identity
- It enables both human and machine access patterns across AWS

## Related Services

- [[aws-vpc]] - Network boundaries complement identity controls
- IAM Identity Center - Workforce access across accounts
- AWS Organizations - Multi-account governance and guardrails
